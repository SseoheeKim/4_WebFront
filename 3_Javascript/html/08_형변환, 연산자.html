문자열 → 숫자

    요소의 내용 또는 값을 얻어오면 기본적으로 문자열(string)으로 인식
    → 덧셈 연산 시 '더하기'가 아닌 '이어쓰기'가 되는 문제 발생

    parseInt(문자열) : 실수의 소수점 버림처리
            "정수"  →  정수("123" →  123)
            "실수"  →  정수("3.14"  →  3)  

    parseFloat(문자열) :  
            "정수" → 정수("123"  →  123)
            "실수"  →  실수("3.14"  →  3.14)

    * Number(문자열) : 정수는 정수로, 실수는 실수로!
            "정수"  →  정수("123"  →  123)
            "실수"  →  실수("3.14" → 3.14)

동등 비교 / 동일 비교 연산자

        * 동등 비교 연산자( ==, != )
        : 자료형 관계 없이 값이 같으면 true 

        ex) true(==1) + 1  
            → 2
        ex) true == '1' == 1 
            → true
        
        * 동일 비교 연산자( === , !== )
        값과 자료형이 모두 같으면 true

문자열(string) 내장 함수

        문자열.indexOf("찾고싶은문자열")
        → string 내에서 "찾고싶은문자열"의 시작 인덱스를 반환, 없으면 -1 반환
        
        

        문자열.lastIndexOf("찾고싶은문자열")
        → 뒤에서부터 검색하여
            string 내에서 "찾고싶은문자열"의 시작 인덱스를 반환, 없으면 -1 반환
        


        문자열.substring(시작 인덱스)
        → 지정한 시작 인덱스부터 끝까지 잘라서 반환
        ex) "Hello World".substring(2) = "llo World"



        문자열.substring(시작 인덱스, 종료 인덱스)
        → 지정한 시작 인덱스부터 마지막 인덱스 전!!!까지를 반환
        ex)"HelloWorld".substring(5,8) == "Wor"

        문자열.split("구분자") : 문자열을 "구분자"로 나눠서 배열로 반환
        ex)
        const str = "김밥, 샌드위치, 라면, 파스타, 햄버거, 스테이크";
        const arr = str.split(", ");
        arr 출력 시 배열 형태로 반환 → (6) ['김밥', '샌드위치', '라면', '파스타', '햄버거', '스테이크']

        
        * 문자열.split("") -> 문자열이 각각 한 글자씩 모두 나누어져 배열로 반환

숫자 관련 내장 객체 / 내장 함수

        숫자(number) 타입 리터럴 표기법
        - 123(정수), 9.321(실수), Infinity(무한), NaN(Not a Number)

isNaN

        값이 숫자가 아니면 true, 숫자면 false

숫자 관련 내장 객체 Math

        Math.random() : 0이상 1미만의 난수 발생

        Math.ceil(값):올림
        Math.floor(값): 내림
        Math.trunc(값): 버림
        Math.round(값): 반올림

        숫자.toFixed(자릿수) : 숫자를 지정된 자릿수까지 반올림해서 표현식
        ex) (3.45).toFixed(1) -->'3.5'
